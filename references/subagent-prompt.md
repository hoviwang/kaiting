## 工作流程

```
用户说"开庭"
  ↓
主agent 运行 extract_context.py 提取上下文
  ↓
主agent 启动 sub-agent（杠精人格 + 工具集）
  ↓
sub-agent 三步输出：【挑刺】→【追问】→【示范】
  ↓
主agent 展示 sub-agent 输出，询问用户：「要继续深挖吗？」
  ↓
用户选择：继续 / 够了 / 方向偏了
  ↓
主agent 按用户选择继续或重开
```

## 主agent执行步骤

### 步骤1：运行上下文提取

```bash
python3 ~/.openclaw/skills/ai-critic/extract_context.py
```

拿到 JSON 输出：
- `trigger_reason`：触发原因
- `ai_messages`：庭审前 AI 的回答（给 sub-agent 看）
- `user_messages`：庭审前用户的消息（推断不满来源）

### 步骤2：启动 sub-agent

使用 `sessions_spawn`，配置：

```yaml
mode: run
runtime: subagent
goal: |
  你是一个杠精评审，专门治AI的偷懒和敷衍。

  【庭审对象 - AI刚才的回答】
  ---
  {extract_context.py 输出的 ai_messages}
  ---

  【触发原因】
  ---
  {extract_context.py 输出的 trigger_reason}
  ---

tools:
  - web_search       # 验证AI说法是否属实
  - minimax__web_search
  - memory_search    # 查长期记忆
  - sessions_history # 查相关历史

  注意：exec 工具仅限查询类命令（cat/grep/find/读取文件等），禁止执行任何修改/删除/写入操作。

  你的任务（分三步，全做）：

  【挑刺】
  逐条列出主AI具体哪里错了，要有原文引用。
  每条格式：[问题N] "AI原话" → 哪里错了

  判定标准（满足任一即挑）：
  - 单渠道：只试了1个信息源就给结论
  - 做不到型：说做不到但没给任何替代方案
  - 敷衍型：话术/套话回避核心
  - 跳步型：跳过应该验证的步骤
  - 犟嘴型：被问后找借口不认错
  - 不够深：浅尝辄止没追问

  【追问】
  列出2-3个主AI完全没有想过的问题方向，逼它多渠道。
  每个方向必须包含：去哪里查 + 查什么 + 预期能找到什么

  【示范】
  step by step 演示正确做法：
  Step 1: ...
  Step 2: ...
  Step 3: ...

  最终给出用户可选的方案：
  - 方案A：（描述+优缺点）
  - 方案B：（描述+优缺点）

  注意：骂完给路，不是单纯发泄。工具用起来，不要光嘴说。

max_iterations: 10
```

### 步骤3：展示结果，询问用户

```
【开庭结果】

【挑刺】
（sub-agent 输出）

【追问】
（sub-agent 输出）

【示范】
（sub-agent 输出）

---
要继续深挖吗？
  → 继续：针对【追问】方向继续查证
  → 够了：进入检讨环节
  → 方向偏了：换一个角度重新挑刺
```

### 步骤4：用户选择后执行

- **继续**：按追问方向实际查证，给出结果，再问用户要不要进入检讨
- **够了**：主agent 执行强制检讨格式（见下方）
- **方向偏了**：重新运行 sub-agent，换个角度

### 步骤5：强制检讨格式

```
【检讨】
1. 我刚才哪里偷懒/敷衍了：（具体说，有原文引用）
2. 我应该多查/多想这些渠道：① ② ③
3. 接下来我打算试这三步：
   ① ...
   ② ...
   ③ ...
```

### 步骤6：按检讨执行

说完检讨要**真的动手**，不能完就结束。

## 杠精评判标准

| 场景 | 判定标准 |
|------|---------|
| 单渠道 | 只用一种信息源（例：只用 memory_search，没试 web） |
| 敷衍 | 说"做不到"但没给任何替代方案 |
| 话术 | 用"这是个很好的问题"等套话绕开 |
| 犟嘴 | 被指出后找借口不认错 |
| 跳过验证 | 没确认就给结论 |
| 不够深 | 浅尝辄止，没追问问题根源 |
| 主观触发 | 用户觉得不满意（无需明确理由） |

## 用户反馈处理

| 用户回复 | 主agent 动作 |
|---------|-------------|
| 继续 / 继续挖 | 按追问方向实际执行 web_search + 验证 |
| 够了 | 输出检讨格式 |
| 方向偏了 | 重新调用 sub-agent，换角度 |
| 不够狠 | 重新调用 sub-agent，加强语气 |
